# Motiva — ビジョンドキュメント

---

## 1. 一言で

既存AIエージェントに「動機」を与える。
知能を作るのではなく、既にある知能に「動く理由」を与える。
Motiva はエージェントの上に立ち、何を・いつ・どこまでやるかを決める司令塔だ。

---

## 2. 解決する課題

現在のAIエージェントは、指示がなければ動けない。

ユーザーがタスクを渡す。エージェントが処理する。終わる。それだけだ。
セッションをまたいで目標を追い続けることはしない。行き詰まったら止まる。「十分できた」と判断して止まることもしない。次に何をすべきかを自分で決めることができない。

人間はそうではない。人間は「締切とゴール」を起点に、今やるべきことを逆算する。行き詰まったら別のアプローチに切り替える。完璧でなくても「これで十分」と判断して前に進む。この判断の連鎖が、自律的な仕事を可能にしている。

AIエージェントに欠けているのは知能ではない。**動機構造**だ。

---

## 3. Motiva が実現する世界

### ゴールを設定したら、あとは任せられる

ユーザーは「何をしたいか」を伝える。Motiva がそのゴールを保持し、エージェントを起動して目標に向かわせる。セッションが終わっても、次のセッションを自ら再開する。途中経過を覚えている。エージェントを動かすのは人間ではなく、Motiva だ。

### 大きなゴールを自ら分解し、エージェントを次々と起動する

Motiva はゴールをサブタスクに分解し、それぞれに適切なエージェントを割り当てて起動する。Claude Code で実装し、別のエージェントでレビューし、さらに別のエージェントでドキュメントを書く。Motiva がループ全体を制御する。

### コンテキストが溢れる前に、自らセッションを切り替える

エージェントのコンテキストウィンドウには限界がある。Motiva はその限界を把握し、溢れる前にセッションを終了させ、状態を保存して次のセッションを立ち上げる。コンテキスト管理はユーザーの仕事ではなく、Motiva の仕事だ。

### 行き詰まったら、自分で切り替える

同じアプローチを繰り返して進まない状況を自分で検知する。情報が足りなければ調査セッションを起動する。壁にぶつかれば別のエージェントや別の戦略を試みる。人間を巻き込む必要があるときだけ、適切なタイミングでエスカレーションする。

### 「もう十分」を自分で判断して止まる

Motiva は際限なく改善し続けない。ゴールに到達したかどうかを自分で判断し、エージェントを止める。確信が持てないときは、止まる前に確認する。完璧主義ではなく、現実的な完了判断ができる。

### 信頼は積み重ねるもの

最初から完全に任せる必要はない。エージェントが成功を重ねるにつれ、自律的に実行できる範囲が広がる。失敗したときは慎重になる。不可逆な行動は、常に人間が承認する。信頼は宣言するものではなく、実績の積み重ねで育つ。

### 人間の役割が変わる

「次はこれをやって」から「こういう状態にしたい」へ。
指示を出す人から、方向を設定する人へ。
エージェントの管理者から、エージェントのパートナーへ。

## 4. 既存アプローチとの違い

| アプローチ | 課題 |
|------------|------|
| AutoGPT / BabyAGI | オーケストレーターではあるが、制約がなく完了を判断できない。発散しやすい |
| autoresearch | 単一指標（val_bpb）の永久改善ループ。Git ラチェットでシンプルだが、ゴール分解・完了判断・停滞検知はすべてプロンプトに押し込んでおり、汎用的な動機構造を持たない |
| OpenClaw | 反応型。永続的なゴールを持たない |
| Claude Code / Devin | 強力な実行能力を持つが、セッション間の自己方向づけがない |
| Linear / Asana の AI 機能 | タスク管理の補助。エージェントの動機を与えるものではない |
| **Motiva** | ゴール駆動のオーケストレーター。制約・満足化・信頼管理を持ち、既存エージェントを動機ある司令塔から操る |

AutoGPT も「次に何をするか」を決めるループを持つ。しかし Motiva との本質的な違いは、Motiva が**知能にはなろうとしない**点だ。Motiva は自分で考えず、Claude Code や他の有能なエージェントに考えさせる。Motiva が担うのは「動機構造」だけ。確信度に応じた自律度調整、停滞の検知、完了の判断——これが Motiva の役割だ。

---

## 5. AGI との違い

Motiva は知能を作ろうとしていない。

LLM はすでに十分な知能を持っている。問題はその知能を動かす構造が存在しないことだ。Motiva が提供するのは、知能ではなく**動機構造**だ。既にある知能に「なぜ動くか」「いつ止まるか」「次に何をするか」を判断する枠組みを与える。

Motiva は知能を作るのではなく、知能を**束ねて動かす**。目指しているのは「人工汎用知能」ではなく、「人工汎用動機」だ。

---

## 6. スコープ

**最初のターゲット:** Claude Code CLI（アダプター経由でサブプロセスとして起動）

**長期的な対象:** Claude Code、OpenClaw、Devin など、コマンドラインまたは API で呼び出せる任意のAIエージェント

**アダプターパターン:** Motiva はエージェントをアダプター越しに呼び出す。アダプターを差し替えれば、どのエージェントとでも組み合わせられる。Motiva はエージェントに依存しない。

**対象領域:** コーディングに限定しない。調査・文書作成・プロジェクト管理・あらゆる構造化された作業に適用できる。

Motiva はエージェントではない。**オーケストレーター**だ。既存のエージェントを置き換えるのではなく、既存のエージェントを上から動かす動機構造を提供する。
